Create a project

This first step to start working with SKY UX is to create a project for a single-page application.

  1. From the command prompt, cd into the directory where you want to create the project.

  2. Enter skyux new. A series of prompts appears.

  3. In the first prompt, specify a root directory for your project. The SKY UX CLI will create a folder with this name and prefix it with "skyux-spa."

    The root directory determines how to access the SPA because the URL for the home page will include the root directory after the base URL.

  4. In the second prompt, specify the URL for your Git repo. You can use a Visual Studio Team System Git repo or a GitHub repo. The URL indicates where to store the source files for your project. The SKY UX CLI will clone the repo and automatically pull in the SKY UX template.

    The only special requirement for the Git repo to host internal Blackbaud implementations is web hooks to the SKY UX continuous integration layer. For now, this must be set up manually by the SKY UX team. Contact the SKY UX team on the 1bb-skyux-apps Slack channel. For more information, see the overview of the continuous integration layer in the overview documentation.

    If your Git repo is not available when you create a SKY UX project, you can skip the second prompt and work on your project locally. Then after the repo is ready, you can manually add the repo as a remote to your local version. In the command prompt, you enter git remote add origin [URL], where "[URL]" is the path for your Visual Studio Team System or GitHub repo. Then to push your local content to the external repo, you enter git push origin --all.

  5. After the SKY UX CLI downloads the SKY UX template project and automatically installs its dependencies, the command prompt displays the directory name: "SPA [directory name] created in directory "skyux-spa-[directory name]."

    For information about the default items that the template provides in a SKY UX single-page application, see the template items reference.
