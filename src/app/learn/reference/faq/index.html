This section provides answers to some frequently asked questions about SKY UX 2.

 How do I get the latest version of the SKY UX CLI?

The command to get the latest is npm install -g @blackbaud/skyux-cli@latest. The SKY UX CLI notifies users when a new version is released. If a new version is available when users run skyux serve, then a message appears in the command prompt.

How do I get the latest version of the SKY UX components?

In the package.json configuration file for NPM that the SKY UX template provides in the default SPA, update the @blackbaud/skyux property that specifies the version of SKY UX components. To find the latest version of the SKY UX components, check the changelog in the SKY UX 2 repo or the NPM page for the SKY UX 2 package.

How do I get the latest version of the SKY UX Builder?

In the package.json configuration file for NPM that the SKY UX template provides in the default SPA, update the @blackbaud/skyux-builder that specifies the version of SKY UX Builder. To find the latest version of SKY UX Builder, check the changelog in the SKY UX Builder repo or the NPM page for the SKY UX Builder package.

What is the timeline for SKY UX 2 to be production-ready?

SKY UX components are production-ready now. We are still in beta because not every component has been moved over and because we still need to create documentation.

For internal Blackbaud SPAs, will deployment be automated through the skyux-deploy repo?

Yes. Currently, any merge to your master branch triggers a release to production. We plan to change that to state a release with an endpoint or a user interface where you can flag staged releases as production releases.

Do you need a separate repo for a backend to perform business logic?

Since the app repository exists solely for UI components, you do need a separate repo if you need a backend deployed to perform business logic or some other action.
