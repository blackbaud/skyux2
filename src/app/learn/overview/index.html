SKY UX 2 Overview

The SKY UX framework provides a consistent, cohesive user experience for Blackbaud products and for third-party developers. SKY UX provides the components, services, and modules that developers need to incorporate Blackbaud styles and patterns.

And with SKY UX 2, the framework now provides additional benefits. First, the SKY UX Builder developer tool now abstracts many of the complexities of modern web development. SKY UX Builder takes the application-building aspects of Angular 2 and wraps them up. We still expect developers to write JavaScript/Typescript and build features, but we don’t expect them to handle all the underlying details. In addition, SKY UX 2 includes a continuous integration layer for testing and validation, and SKY UX Host provides as a hosting solution for Blackbaud applications.

What's new in SKY UX 2

SKY UX 2 continues to provide many of the same benefits as the first version of SKY UX. It continues to provide design patterns to implement a consistent, cohesive user experience and guidance on how to apply components and the context for when to use them. It also continues to provide Angular components, services, and modules to create user interfaces that incorporate SKY UX styles and patterns.

In addition, SKY UX 2 includes the following improvements and new capabilities:

  • SKY UX 2 takes advantage of Angular 2 to keep SKY UX modern while increasing the tooling, testing, and performance available.

  • SKY UX Builder provides for a rapid development process with minimal developer setup. It creates single-page applications and handles configuration details for developers.

  • A continuous integration layer runs unit tests, visual regression tests, and end-to-end tests in multiple platforms and browsers when an SPA's code changes or when an SPA dependency releases a new version. This layer also stages the SPA for additional testing and validation.

  • SKY UX Host acts as a gateway for all requests to SKY UX SPAs. When serving locally, it pulls files locally. And for production, it pulls files from a registry that stores static files.

SKY UX Builder

SKY UX Builder is cross-platform command line developer tool to create, test, and run SPAs. It abstracts most of the tedious complexities of modern web development, including bundling configuration, unit test configuration, and routing/module configuration. This allows developers to focus on features, not infrastructure. And it allows SPAs to be developed, tested, deployed, and hosted in a consistent way across an entire application.

  • Handles details such as transpilation, linting, unit testing, and bundling/chunking/minifying.

  • By default, uses automatic route configuration based on a project's folder structure. This means that you must place files in specific locations to make sure that they are recognized. For example, to create a page, you add a folder in the existing \src\app folder and add an index.html file to that new folder.

  • Provides build and testing configurations. This includes settings to use Webpack to handle minification and build JavaScript assets, Karma unit tests, and Protractor end-to-end tests.

  • Allows for easy implementation of Blackbaud Authentication.

Continuous integration layer

The continuous integration layer of SKY UX runs unit tests, visual regression tests, and e2e tests in multiple platforms and browsers when the SPA code changes or when an SPA dependency releases a new version. It also handles staging the SPA for additional testing and validation.

SKY UX Host

SKY UX Host is the gateway for requests to SKY UX SPAs. When serving locally, SKY UX Host pulls files locally. And for production, SKY UX Host pulls files from a registry where SKY UX Builder stores them. When the builder runs in the continuous integration layer, it stores the static files that it produces (JavaScript, JSON, etc.) in the registry for SKY UX Host to serve them.

SKY UX Host creates a web service to handle the following details:

  • Serve up an HTML page to host the SPA.

  • Set the appropriate cache headers for the HTML page and static files.

  • Set the appropriate handlers to support HTML 5-style routing.

  • Serve as an origin for a content delivery network (CDN).

  • Register JavaScript and CSS for the SPA. This includes registering locale files based on browser culture, using sub-resource integrity checking for CDN-hosted files, and using a fallback strategy for when the CDN is unavailable.

  • Implement HTTP Strict Transport Security (HSTS).

  • Obtain, configure, and deploy SSL certificates.

  • Maintain highly availability through load-balancing, georeplication, and monitoring.
