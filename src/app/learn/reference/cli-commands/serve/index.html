skyux serve serves the current SKY UX single-page application locally. This command also opens the host endpoint with the necessary query string to pull your local files. It tacks on local=true in the query string to instruct SKY UX Host to pull files from the local machine instead of the hosted location. By default, the SPA automatically opens in the default browser, but you can specify a different browser with the --browser option. Or you can add the --launch none option to prevent the SPA from opening. The serve command basically updates the production location rather than creating a duplicate local instance, so to view the production version alongside a local version, you need to open a separate incognito tab. The serve command sets a cookie to apply local dev mode. Keep in mind that since skyux serve serves your single-page application over HTTPS, any additional requests from your code such as AJAX requests must also be over HTTPS.

If you serve in Internet Explorer 11 or Microsoft Edge, the default browser settings can cause the SPA to reload indefinitely. To fix the browser settings, open the Control Panel from the Start menu and select Internet Options. On the Internet Properties screen, go to the Security tab and click Local intranet. Then click Sites, and on the Local intranet screen, clear Include all local (intranet) sites not listed in other zones. Click OK to save the change and then click OK on the Internet Properties screen. For more information, see the issue in the SKY UX repo. Options
