Edit an existing page

After you serve the default single-page application, you can update an existing page.

  1. With skyux serve running, navigate to the default site's About page in your web browser.

  2. In a local editor, open the project's about.component.html file in the src/app/about folder.

    About our Team

    Warning! We never check our email.
    {{ member.name }} {{ member.email }}

  3. At the top of the file before the warning, add a paragraph such as

    This page provides contact information for members of our team.

  4. Save the changes.

  5. Return to the command prompt to see that skyux serve recognizes the changes and updates the page.

  6. Return to the About page in the web browser to see the new paragraph in place.

Edit a component on an existing page

After you edit an existing page in the default single-page application, you can update an existing component on the page.

  1. With skyux serve running, return to the default site's About page in your web browser. The paragraph from the previous tutorial should be in place.

  2. In a local editor, return to the project’s about.component.html file.

  3. Edit the type and message for the alert component. For example, in the alertType attribute, change the value to from warning to info. And in the sky-alert element, change the message to There may be a delay in responding to your email.

    Note: There may be a delay in responding to your email.

  4. Save the changes.

  5. Return to the command prompt to see that skyux serve recognizes the changes and updates the page.

  6. Return to the About page in the web browser to see that the new alertType value updates the background color and that the new text replaces the previous message.

Add a component to an existing page

After you edit a component on an existing page in the default single-page application, you can add a SKY UX component to the page.

  1. With skyux serve running, return to the default site's About page in your web browser. The updates to the alert component from the previous tutorial should be in place.

  2. In a local editor, return to the project’s about.component.html file.

  3. Between the alert and avatar components, add a card component. Add the sky-card element, and then within that element, add the sky-card-title element for the header text, the sky-card-content element for the body content, and the sky-card-actions element for the button.

    Large card This card demonstrates the amount of space that is available for a card that uses the default large size. If the content does not require this much space, you can set the card size to small. The type of content to display in the body of a card varies based on what the card represents and whether it prompts users to action. The content can consist of some combination of a graph, trend, status, summary information, and guidance text.

  4. Save the changes.

  5. Return to the command prompt to see that skyux serve recognizes the changes and updates the page.

  6. Return to the About page in the web browser to see the new card component between the alert and avatar components.

Create a page

After you add a SKY UX component to an existing page in the default single-page application, you can create a page. By default, SKY UX uses automatic route configuration based on the project's folder structure. So to create a page, you simply create a folder in the existing src/app folder and then create an index.html file in that folder.

  1. Return to the project in a local editor, and create a folder within the app folder.

  2. Rename the folder. The folder name appears in the route for the page. For this tutorial, use "demo" for the folder name.

  3. In the new demo folder, create a file and name it index.html.

  4. In the new file, place the shared app-nav component at the top and then add hello world content such as <h1>Hello world!</h1>. The app-nav component is an example of a shared component. It manually creates top-level navigation within your app.

    Hello World!

  5. Save the changes to the index.html file.

  6. Go to the shared folder, and open the app-nav.component.ts file.

  7. Add a path for the new page with name set to "Demo" and path set to "/demo." The name provides the label for the menu, and the path specifies the name of the folder.

    import { Component } from '@angular/core'; @Component({ selector: 'app-nav', templateUrl: './app-nav.component.html', styleUrls: ['./app-nav.component.scss'] }) export class AppNavComponent { public nav = [ { name: 'Home', path: '/' }, { name: 'About', path: '/about' }, { name: 'Demo', path: '/demo' } ]; }

  8. Save the changes to the component file.

  9. If skyux serve is running, it recognizes the changes and updates the SPA. If not, open the command prompt and enter skyux serve to launch the SPA.

  10. Return to the default site to see the update in place. The Demo item appears in the menu.

  11. Click the Demo menu option. The new page appears.

Create a component

Now that you've added some basic content to your page, you may need more programmatic control over the contents of the page. For instance, you might want to display some dynamic data based on a web request or some client-side logic written in TypeScript. For that, you need to create your own component.

The SKY UX CLI makes it easy to create components from the command line. For this example, we'll create a component in the demo folder from the previous tutorial.

  1. From the command line, run skyux generate component demo/my-demo in the context of your project's root directory. This creates the TypeScript, HTML, SCSS, and spec files to build and test your component.

  2. In your code editor, navigate to the demo folder, and open the my-demo.component.ts file that includes boilerplate code to create your component.

    import { Component } from '@angular/core'; @Component({ selector: 'app-my-demo', templateUrl: './my-demo.component.html', styleUrls: ['./my-demo.component.scss'] }) export class MyDemoComponent { }
  3. We want our component to accept input from the host page. This requires a public property with the Angular @Input decorator. We can import this decorator from @angular/core. For this example, we'll add an input property called message. import { Component, Input } from '@angular/core'; @Component({ selector: 'app-my-demo', templateUrl: './my-demo.component.html', styleUrls: ['./my-demo.component.scss'] }) export class MyDemoComponent { @Input() public message: string; }
  4. Now that the component accepts input, we can display input in the component's template. Open the my-demo.component.html file, and add the following code to display the message. {{ message }}

    The curly braces are the syntax to reference public properties of the component class.

  5. Open the index.html file in the demo folder, and add the new app-my-demo component. Set the value of the message attribute to "Hello!"

  6. Return to the command prompt, and run skyux serve to launch the SPA in your default browser.

  7. Return to the Demo page in the SPA. You should see "Hello!" input text rendered after the header.
