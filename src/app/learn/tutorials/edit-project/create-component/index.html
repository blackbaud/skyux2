Create a component

After you create a page in the default single-page application, you can create a component for the page.

  1. Return to the project in a local editor.

  2. In the demo folder from the previous tutorial, add a new file and name it demo.component.ts. This file will define the component and its functionality.

    If skyux serve is running, it detects the new file and attempts to update the SPA. However, errors occur when trying to serve the empty file, and the SPA fails to compile. In the command prompt, you can see the errors.

  3. Add another file in the demo folder and name it demo.component.html. This file will be the template for the new component to use.

  4. In the demo.component.ts file, add the following code for a component to display a message on the Demo page.

    
            import {
                Component,
                Input
            } from '@angular/core';
            
            @Component({
                selector: 'my-demo',
                templateUrl: 'demo.component.html'
            })
            export class MyDemoComponent {
                @Input()
                public message: string;
            }

    The import portion imports the Component and Input classes from Angular to provide that functionality to the component that we are defining. The @Component section is a decorator for the component, and it defines our selector and template. The decorator takes normal JavaScript classes and extends the JavaScript functionality to convert them into Angular components. The export portion defines the component class and a public input variable called message.

    If your component requires a .scss file to extend the CSS, you can add a .scss file in the same location as the component files and use the styleUrls property in the @Component section to define the .scss file.

  5. In the demo.component.html file, enter {{'{{'}} message }} to display the message. This provides the message variable to the template. The curly braces are the syntax to reference properties of the component class.

  6. Open the index.html file in the demo folder, and add the new my-demo component. Set the value of the message variable to "Hello!"

    <my-demo message="Hello!"></my-demo>

  7. Save all three files.

  8. Return to the command prompt. If skyux serve is running, it detects the changes and updates the SPA. If not, run skyux serve to launch the SPA in your default browser.

  9. Return to the Demo page in the SPA. It renders the text from the component after the header.
