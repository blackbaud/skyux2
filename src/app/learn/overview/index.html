SKY UX provides a consistent, cohesive user experience for Blackbaud products. The framework includes the components, services, and modules that developers need to incorporate Blackbaud styles and patterns.

Starting with SKY UX 2, the framework also provides additional benefits. The SKY UX Builder developer tool takes the application-building aspects of Angular and wraps them up to abstract many of the complexities of modern web development. In addition, SKY UX 2 includes a continuous integration layer for testing and validation, and SKY UX Host provides a hosting solution for Blackbaud applications.

Keep in mind that Angular is at the heart of SKY UX, so developers need to be familiar with this technology to build with SKY UX. For training materials on Angular and other SKY UX tools and technologies, see the training resources. What's new in SKY UX 2

SKY UX 2 continues to provide the same benefits as the previous version of SKY UX: Its Angular components, services, and modules enable developers to create user interfaces that incorporate SKY UX styles and patterns. SKY UX design patterns allow developers to implement a consistent, cohesive user experience and provide guidance on how to apply components and the context for when to use them.

In addition, SKY UX 2 also includes the following improvements and new capabilities:

Components, services, and modules are open source and available to external developers in the SKY UX 2 GitHub repo, but some SKY UX 2 features are for internal Blackbaud use only.

  • SKY UX 2 takes advantage of Angular 2 to keep SKY UX modern while increasing the tooling, testing, and performance available. Components, services, and modules are open source in the SKY UX 2 GitHub repo.

  • SKY UX Builder provides for a rapid development process with minimal developer setup. It creates single-page applications and handles configuration details for developers.

  • The continuous integration layer runs unit tests, visual regression tests, and end-to-end tests in multiple platforms and browsers when a SPA's code changes or when a dependency releases a new version. This layer also stages the SPA for additional testing and validation.

  • SKY UX Host acts as a gateway for all requests to SKY UX SPAs. When serving locally, it pulls files locally. And for production, it pulls files from a registry that stores static files.

SKY UX Builder

SKY UX Builder is cross-platform, command line developer tool to create, test, and run SPAs. It abstracts most of the tedious complexities of modern web development, including bundling configuration, unit test configuration, and routing/module configuration. This allows developers to focus on features, not infrastructure. And it enables developers to build, test, deploy, and host SPAs in a consistent way across an entire application.

External developers can use Builder to create, test, and run SKY UX SPAs, but some features are for internal Blackbaud use only.

In addition, SKY UX Builder also:

  • Handles details such as transpilation, linting, unit testing, and bundling/chunking/minifying.

  • Uses automatic route configuration by default based on a project's folder structure. This means you must place files in specific locations to make sure that they are recognized. For example, to create a page, you create a folder in the existing \src\app folder and then create an index.html file in that folder. For more information about where to place files, see the file name and placement reference.

  • Provides build and testing configurations. This includes settings to use Webpack to handle minification and build JavaScript assets, Karma unit tests, and Protractor end-to-end tests.

  • Allows for easy implementation of Blackbaud Authentication.

Continuous integration layer

The continuous integration layer of SKY UX runs unit tests, visual regression tests, and end-to-end tests in multiple platforms and browsers when SPA code changes or when a dependency releases a new version. It also handles staging SPAs for additional testing and validation.

The CI layer is primarily for internal Blackbaud use. External developers can use this layer when developing SPAs locally, but it is not open source like the components, services, and modules in the SKY UX 2 GitHub repo. SKY UX Host

SKY UX Host is the gateway for requests to SKY UX SPAs. During local development, SKY UX Host loads local files. And for production, SKY UX Host loads bundled files from a CDN where SKY UX Builder stores them.

Host is primarily for internal Blackbaud use. When external developers use the SKY UX CLI to serve SKY UX SPAs locally, Host loads local files. However, Host is not open source like the components, services, and modules in the SKY UX 2 GitHub repo.

SKY UX Host also creates a web service that:

  • Serves up an HTML page to host the SPA.

  • Sets the appropriate cache headers for the HTML page and static files.

  • Sets the appropriate handlers to support HTML 5-style routing.

  • Serves as an origin for a content delivery network (CDN).

  • Registers JavaScript and CSS for the SPA. This includes registering locale files based on browser culture, using sub-resource integrity to check for CDN-hosted files, and using a fallback strategy for when the CDN is unavailable.

  • Implements HTTP Strict Transport Security (HSTS).

  • Obtains, configures, and deploys SSL certificates.

  • Maintains high availability through load-balancing, georeplication, and monitoring.
