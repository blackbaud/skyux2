This reference describes the helpers that are available to make authenticated HTTP requests.

External developers should keep in mind that some SKY UX options described in the technical references are for internal Blackbaud use only. External developers can take advantage of SKY UX to create, test, and run SKY UX SPAs, and the SKY UX components, services, and modules are open source and available in the SKY UX 2 GitHub repo. However, not all SKY UX features are available for external use.

  • import { SkyAuthHttp } from '@blackbaud/skyux-builder/runtime — In conjunction with the "auth": true configuration setting in the skyuxconfig.json file, we provide a wrapper to the http utility class. Use SkyAuthHttp.get() to automatically add the necessary Authorization header containing the bearer token received from authentication. This helper is for internal Blackbaud use only.

  • import { SkyHostBrowser } from '@blackbaud/skyux-builder/runtime/testing — Particularly useful when running the skyux e2e command in the SKY UX CLI. Use SkyHostBrowser.get() to automatically add the SKY UX Host base URL, as well as the necessary configuration object to enable local development mode. For example, SkyHostBrowser.get(/my-page?my-qs=1');.
