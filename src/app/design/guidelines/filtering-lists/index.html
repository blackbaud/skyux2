SKY UX provides three filter patterns to accommodate different filter scenarios.

 Persistent inline filters

Persistent inline filters display controls to the right of tile or list headers. The filter is applied immediately after users make selections. This pattern only supports simple filter controls such as checkbox toggles and dropdowns.

This pattern is appropriate for scenarios where users can only apply one or two simple filter options. Since the filter controls are always visible, users can quickly apply filter options without invoking them through a button.

Expandable filters

Expandable filters hide filter controls by default and allow users to display them inline by clicking "Filter" buttons in tile or list headers. For example, the list component can display a "Filter" button in the list header. After users apply filters, the "Filter" button's text and outline change to green to indicate that the data is filtered.

The filter is applied immediately after users make selections. This pattern only supports simple filter controls such as checkbox toggles and dropdowns.

This pattern is appropriate for scenarios where users can apply up to four simple filter options.

 Filters in modals

Filters in a modal hide filter controls by default and allow users to display them in a modal by clicking "Filter" buttons in tile or list headers. For example, the filter component can display a "Filter" button in a list header. After users apply filters, boxes on the page indicate the applied filters and the "Filter" button's text and outline change to green to indicate that the data is filtered.

Filters in modals can include both simple filter controls and more complex controls.

This pattern is appropriate for scenarios where users can apply more than four simple filter options or where one or more complex filter controls are necessary.
