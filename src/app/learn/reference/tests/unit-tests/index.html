SKY UX automatically configures Karma to run unit tests in the src\app folder that match the .spec.ts pattern. We recommend that you follow the Angular 2 naming guidelines on how to name spec files and where to store them. SKY UX unit tests should use the same name as the components or services that they test, but they should end with .spec.ts. And you place the spec files in the same folder as the components or services that they test. When you run skyux test or skyux watch, code coverage results are stored in the coverage folder.

When you create a SKY UX project, the template includes sample unit tests that you can run to test the default single-page-application. From the command line, you can use the skyux test command to run unit tests and the skyux watch command to run unit tests and watch the file system for changes.

SKY UX styles are loaded when running unit tests, so CSS rules defined in SKY UX take effect during tests. This allows you to check the expected computed style of an element when using things like the HTML hidden property that only take effect when SKY UX styles are loaded.

For tips and techniques on testing Angular applications, see the Angular 2 testing guidelines. And for information about how to test modals in SKY UX, see write unit tests for modals.
