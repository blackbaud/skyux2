Template items

This reference describes the default items that the template provides in a SKY UX single-page application.

After you create a SKY UX project, a variety of files and folders are already in place, and some SKY UX CLI commands create additional items in the project. This topic explains the purpose of these files and folders to provide guidance to developers on which items to edit and which items to ignore.

The default items that the SKY UX template provides in a SPA include:

  • .vscode folder — Contains 2 .json files to configure settings that are included for the developer's convenience.

    • launch.json — A launch configuration file that provides settings on how to debug the build process for the SPA.

    • settings.json — A file that controls editor settings for VSCode.

  • coverage folder — Stores code coverage results for unit tests. This folder is generated when you run skyux test or skyux watch, and it provides a convenient way for developers to see what code the tests reached and what code they did not reach. We recommend that all new code should have 100 percent unit test code coverage. For information about the code coverage requirements to contribute to SKY UX, see the SKY UX 2 contribution guidelines.

  • dist folder — Contains files that are deployed through the continuous integration pipeline. This folder is generated when you run skyux build.

  • e2e folder — The required location to store end-to-end tests. As a convenience, this folder contains sample tests.

  • node_modules folder — Stores the external dependencies for SKY UX components and SKY UX Builder. This includes system files to build the single-page application. The folder is listed in the .gitignore file. Do not place it into source code control.

  • src folder — Contains the source files for the SPA.

    • app folder — Contains the files and folders for pages, components, and unit tests.

      • about folder — Contains the page, component, and unit test for the sample About page in the template SPA. The index.html file is required to generate a route, while the other .html file and the .ts file create the AboutComponent that index.html references. The .spec.ts file is the unit test for the component, and we recommend storing the unit test in the same location as the .ts file.

      • shared folder — The recommended location for components and services that are shared across an application. In the template project, the AppNavComponent creates the navigation menu.

      • app-extras.module.ts — Handles tasks such as registering providers and entry components. When this file exists, SKY UX Builder automatically imports it into the build process. For an example of how to register an entry component, see add an entry component.

      • home.component.html, home.component.ts, home.component.spec.ts, and index.html — The page, component, and unit test for the sample Home page in the template SPA. The index.html file is required to generate a route, while the other .html file and the .ts file create the HomeComponent that index.html references. The .spec.ts file is the unit test for the component, and we recommend storing the unit test in the same location as the .ts file.

  • .editorconfig — A configuration file to specify settings for editors that support an EditorConfig file to define and maintain consistent coding styles between editors and IDEs. For information about EditorConfig files, see the EditorConfig website.

  • .gitignore — A standard file for Git repositories that lists untracked files to intentionally ignore in the working directory. For information about gitignore files, see the Git documentation.

  • package.json — A configuration file for NPM to specify dependencies that are stored in the node_modules folder. For information about package.json files, see the NPM documentation.

  • README.md — A Markdown file for the repo to describe the SKY UX application.

  • skyuxconfig.json — A configuration file to specify settings for SKY UX applications. For information about these options, see the configuration options reference.

  • tsconfig.json — A configuration file for the TypeScript compiler. For information about tsconfig.json files, see the TypeScript documentation.

  • tslint.json — A configuration file for TSLint. For information about tslint.json files, see the TSLint documentation.
