The appSettings property in skyuxconfig.json enables you to reuse data throughout a SPA. The appSettings property accepts strings, arrays, and any other type of configuration setting that is necessary. Use these instructions to access the appSettings data throughout your application.

 If you create files to apply configuration settings to specific SKY UX CLI commands, you can provide command-specific appSettings data in those configuration files. Reference appSettings in an index.html file

  1. In a local editor, open the project's skyuxconfig.json file in the root directory.

  2. Insert the appSettings property and specify the data to reuse throughout the application. For example:

    "appSettings": { "exampleSetting1": "foobar" }

  3. Open the index.html file where you want to reference the appSettings value.

  4. Pull in the config service and reference the appSetting value to access it in the index.html file. For example:

    {{ config.skyux.appSettings.exampleSetting1 }}

  5. Save your changes. When you build or serve your application, the appSettings value appears on the page for the index.html file.

Reference appSettings in component or service files

  1. In a local editor, open the project's skyuxconfig.json file in the root directory.

  2. Insert the appSettings property and specify the data to reuse throughout the application. For example:

    "appSettings": { "exampleSetting2": "barfoo" }

  3. Open the component or service files where you want to reference the appSettings value.

  4. In the component.ts file for the component or the service.ts file for the service, import SkyAppConfig from @blackbaud/skyux-builder/runtime:

    import { SkyAppConfig } from '@blackbaud/skyux-builder/runtime';

  5. Add SkyAppConfig as a private property in the constructor for the component or service: constructor(private skyAppConfig: SkyAppConfig) {}.

    import { Component } from '@angular/core'; import { SkyAppConfig } from '@blackbaud/skyux-builder/runtime'; @Component({ selector: 'my-home', templateUrl: './home.component.html' }) export class HomeComponent { constructor(private skyAppConfig: SkyAppConfig) {} }

  6. In the template file for the component or service, pull in the config service and reference the appSettings value to access it in the .html file. For example:

    7. <p>Sample text: {{ skyAppConfig.skyux.appSettings.exampleSetting2 }}</p>

  7. Save your changes. The appSettings value is now available to the component or service.
