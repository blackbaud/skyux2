After you create a SKY UX project, serve the default single-page application, edit pages and components in the project, and run tests for the project, you can include a static asset such as an image in in HTML, CSS, and SCSS files.

  1. Return to the project in a local editor, and create an assets folder in the src directory.

  2. Place a static asset in the src/assets folder. For this tutorial, we’ll add an example.jpg file to the folder.

  3. Return to the project’s about.component.html file.

  4. Between the card and avatar components, add a p element.

  5. Within the p element, add an img element.

  6. In the img element, add a src attribute to map the element to the example.jpg file in src/assets and place ~/assets/ before the filename.

    You must reference static assets with ~/assets/ before the filename because the filenames are updated at build time to include hashes that represent the content of the files.

  7. Save the changes.

  8. If skyux serve is running, it recognizes the changes and updates the SPA. If not, open the command prompt and enter skyux serve to launch the SPA.

  9. Return to the About page in the web browser to see the new image between the card and avatar components.

    For more information about static assets, see the src/assets entry in the file name and placement reference.
