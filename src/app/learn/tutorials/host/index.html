Host a project

After you create a SKY UX project, serve the default single-page application, edit pages and components in the project, and run tests for the project, you can follow the steps in this tutorial to host an internal Blackbaud SPA.

SKY UX Host is the gateway for requests to SKY UX SPAs. During local development, SKY UX Host loads local files. And for production, SKY UX Host loads bundled files from a CDN where SKY UX Builder stores them. For more information, see the SKY UX Host section of the overview. When Builder runs in the continuous integration layer, it stores the static files that it produces (JavaScript, JSON, etc.) in the registry for SKY UX Host to serve them.

  1. If you do not have a Git repo for your project, create a repo in Visual Studio Team System or in Blackbaud's private GitHub account.

  2. Manually add the Git repo as a remote to your local version. In the command prompt, enter git remote add origin [URL], where "[URL]" is the path for your GitHub or VSTS repo.

  3. Push your local content to the external repo. In the command prompt, enter git push origin --all.

  4. Contact the SKY UX team with with the name and location of the repo. Contact @jonathan.bell directly on the 1bb-skyux-apps Slack channel to request web hooks from your Git repo to the SKY UX continuous integration layer of SKY UX. The web hooks are necessary to host internal Blackbaud implementations, and for now, they must be set up manually. The team sets up a build/release that listens to the repo through web hooks and then pushes code to Host.
