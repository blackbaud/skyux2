SKY UX Builder includes localization capabilities that allow you to declare strings in a resources_en_US.json file in src/locales and then reference those strings in your SPA using the skyAppResources pipe or the SkyAppResourcesService service.

  1. In the src/assets/locales folder, open the resources_en_US.json file.

  2. In this file, specify strings for reuse throughout your application. Use JSON structure for the localization strings and specify a sting name, and then provide a description and message for it.

    { "string_name": {} "message": "The localization string", "_description": "A description of the string for the translator." } }

  3. After your localization strings are in place in resources_en_US.json, you can reference them throughout your SPA using the skyAppResources pipe or the SkyAppResourcesService service.

    • To reference localization strings with the skyAppResources pipe, place {{'{{'}} 'string_name' | skyAppResources }} in your remplate to pull the correct string from the JSON file.

    • To reference localization strings with the SkyAppResourcesService service:

      this.resources.getString('string_name') .subscribe((value) => { /*do something with the string*/ });
