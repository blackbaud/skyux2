Initial SKY UX setup

This section provides guidance on how to set up your developer machine for SKY UX 2.

Prerequisites

The first step to set up your machine for SKY UX is to make sure you have the following prerequisites in place:

  • Node.js version 6+ — To check your version, run node –v from the command line.

    Do not use Node.js version 7+.

  • NPM version 3+ — NPM is the default JavaScript package manager for Node.js, and as of Node.js version 0.6.3, it is bundled and installed automatically with the environment. We recommend version 3+ for its flat dependency structure. To check your version, run npm –v from the command line.

  • Git — To confirm that you have Git in place, run git version from the command line.

SSL certificate

During local development with SKY UX, your local files are loaded into SKY UX Host. To load your local single-page application into SKY UX Host, you must install the skyux-ca.crt root certificate.

  • To install the SSL certificate on a Mac, follow these instructions.

  • To install the SSL certificate in Windows, follow these instructions.

Install SKY UX CLI

From the command line, run npm install -g @blackbaud/skyux-cli.

For Mac OS X, we recommend that you use Node Version Manager (nvm) to wrap your NodeJS installation so that it installs in your user directory and avoids permissions-related issues.

Update SKY UX CLI

To update the SKY UX CLI with the latest version, run npm install -g @blackbaud/skyux-cli@latest.

The SKY UX CLI notifies users when a new version is released. If a new version is available when users run skyux serve, then a message appears in the command prompt.
