This section provides guidance on how to set up your developer machine for SKY UX 2.

 Install SKY UX CLI

From the command line, run npm install -g @blackbaud/skyux-cli.

For Mac OS X, we recommend that you use Node Version Manager (nvm) to wrap your NodeJS installation so that it installs in your user directory and avoids permissions-related issues. Update SKY UX CLI

To update the SKY UX CLI with the latest version, run npm install -g @blackbaud/skyux-cli@latest.

The SKY UX CLI notifies users when a new version is released. If a new version is available when users run skyux serve, then a message appears in the command prompt. Update SKY UX components and SKY UX Builder

After you install the SKY UX CLI and create a SKY UX project, the project includes a package.json configuration file for NPM to specify dependencies that are stored in the node_modules folder. The package.json file includes properties to specify versions for SKY UX components and SKY UX Builder.

  • Under the dependencies property, @blackbaud/skyux specifies the version of SKY UX components for the project.

  • Under the devDependencies property, @blackbaud/skyux-builder specifies the version of SKY UX Builder for the project.

To update the version of the components or Builder, manually update the version in package.json and then run npm install.

  • To find the latest version of the SKY UX components, check the changelog in the SKY UX 2 repo or the NPM page for the SKY UX 2 package.

  • To find the latest version of SKY UX Builder, check the changelog in the SKY UX Builder repo or the NPM page for the SKY UX Builder package.

If you use VSCode or another editor with code-editing functionality, you can also place the cursor in the dependencies or devDependencies property and press Ctrl-Space to open a prompt that displays the latest version.
