Configuration options

This reference describes SKY UX configuration options that are available in the skyuxconfig.json file.

After you create a SKY UX project, the skyuxconfig.json file in your project's directory allows you to configure settings for SKY UX. All properties in skyuxconfig.json are optional, and by default, the file only specifies values for the mode and compileMode settings.

The skyuxconfig.json does not include all SKY UX configuration options. SKY UX Builder defines some configuration options that are not included in this file.

The configuration options that are available in the skyuxconfig.json file include:

  • name — Specifies the name of the project when running in SKY UX Host. For example, if you specify "demo," then your SPA will accessible from https://sky.blackbaud-dev.com/demo. By default, SKY UX Builder reads the name from the name property in the package.json file, minus the "blackbaud-skyux-spa-" prefix. This is based on the root directory name that you specify when you create the project. You can overwrite the default name if you do not want the root directory name in your public-facing URL.

  • mode — Controls how much boilerplate code to generate automatically. By default, the skyuxconfig.json file sets this property to easy. In easy mode, SKY UX uses automatic route configuration based on the project's folder structure. To override automatic route configuration, you can change this property to advanced. In advanced mode, you can adjust URLs and point to specific components in your SPA. This property will eventually become obsolete as the SKY UX team builds out the CLI to allow users to override individual steps without switching to advanced mode.

  • compileMode — Specifies how to compile the project's code. By default, the skyuxconfig.json file sets this property to aot to specify ahead-of-time compilation, which compiles high-level language into machine code and executes the resulting binary file natively. You can change this setting to jit to specify just-in-time compilation, which compiles code during run time.

  • host — Specifies configuration options for communication with SKY UX Host.

    • url — Specifies a base URL for passing information from skyux serve to the SKY UX Host. It would be very uncommon to change this. The default value is https://sky.blackbaud-dev.com.

  • app — Specifies configuration options for the local app when running skyux serve.

    • title — Controls the page title. Before Angular2 sets the page title, this property controls the title at the template level. The default is Blackbaud - SKY UX Application. You can also control the title with the title service.

    • externals — Dynamically injects CSS and JS files into SKY UX Host. You should have a specific use-case for an external. For example, Office Addins require their library to load through CDN in the head. The before and after sections of an external indicate whether to include the external resource before or after the default SKY UX Builder resources. The head property, which only applies to JS resources, indicates whether to inject elements within the HTML head element or just before the closing body tag.

      The following example shows an external with all the configuration options:

      externals: {
    css: {
        before: [
            {
                url: 'f1.css',
                integrity: 'ic1'
            }
        ],
        after: [
            {
                url: 'f2.css'
            }
        ]
    },
    js: {
        before: [
            {
                url: 'f1.js',
                integrity: 'ic2',
                head: true
            },
            {
                url: 'f2.js',
                integrity: 'ic3'
            }
        ],
        after: [
            {
                url: 'f3.js'
            }
        ]
    }
}

  • auth — Indicates whether the project requires an authenticated Blackbaud ID. By default, this property is set to false. To require authentication, set this property to true. For information about how to make authenticated HTTP requests, see the helpers reference.

  • omnibar — Specifies an object to pass to the omnibar's load method. This property is for Blackbaud internal use only. For information about available options to pass to the omnibar, see the omnibar configuration options.

  • help — Indicates whether to automatically include the help widget in the project. This property is for Blackbaud internal use only. By default, this property is set to false. To include the help widget, set this property to true.
