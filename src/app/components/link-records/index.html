Usage
Matching certainty

The intent of this component is to allow comparison of records where there is high certainty that a single record is the correct match. For cases where you need to present a set of less-certain possible matches from which the user can choose the correct option you will need a different approach.

Content

To minimize cognitive load the data shown on the comparison cards should be the minimum required to uniquely identify a record and allow for a new record to be created in cases where no true match is found.

Batch processing

Record linking will frequently be done in the context of importing multiple records at once. In these cases the component can be repeated as a vertical list of comparisons as in the demo above. If you are importing multiple types of records the task should be divided by record type such that the user works through multiple lists with each list including a single type of record.
