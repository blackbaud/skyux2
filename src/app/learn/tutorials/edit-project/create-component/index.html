Create a component

This tutorial walks through the process to create a component in a single-page application.

  1. Return to the project in a local editor, and right-click the demo folder we created in the previous tutorial.

  2. Click New File, and then rename the file "demo.component.ts." This file is where we will define the component and its functionality.

    If the skyux serve command is running, it detects the new file and attempts to update the SPA. However, errors occur when trying to serve the empty file, and the SPA fails to compile. If you return to the command prompt, you can see the errors.

  3. Click New File again, and then rename the file "demo.component.html." This file is the template for the newly created component to use.

  4. In the demo.component.ts file, add the following code for a component to display a message on the Demo page.

    
            import {
                Component,
                Input
            } from '@angular/core';
            
            @Component({
                selector: 'my-demo',
                templateUrl: 'demo.component.html'
            })
            export class MyDemoComponent {
                @Input()
                public message: string;
            }

    The import portion imports the Component and Input classes from Angular to provide that functionality to the component that you are defining here. The @Component section is a decorator for the component, and it defines our selector and template. The decorator takes normal JavaScript classes and extends the JavaScript functionality to convert them into Angular components. And finally, the export portion is where we define the component class. In this case, we also define a public input variable called message.

  5. In the demo.component.html file, enter {{'{{'}} message }} to display the message. This provides the message variable to the template. The curly braces are the syntax to reference properties of the component class.

  6. Open the index.html file in the demo folder, and then add the new my-demo component and set the value of the message variable to "Hello!".

    <my-demo message="Hello!"></my-demo>

  7. Save all three files.

  8. Return to the command prompt. If the skyux serve command is running, it detects the changes and updates the SPA. If not, run skyux serve to launch the SPA in your default browser.

  9. Return to the Demo page in the SPA. It now renders the text from the component after the header.
