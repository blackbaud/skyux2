The SKY UX accessibility analyzer tool allows you to run accessibility tests such as aXe automated tests as part of your end-to-end testing. aXe is an open source library of automated accessibility tests that is integrated into SKY UX to identify potential issues for a subset of the Web Content Accessibility Guidelines 2.0 Level A and AA success criteria.

To use the accessibilty analyzer, you add SkyA11y to e2e specs. And to manage rules for accessibility tests, you use the a11y property in the skyuxconfig.json file.

 Add the accessibility analyzer to e2e specs

You can add the accessibility analyzer anywhere in your e2e specs. You manually import the accessibility library and then execute accessibility checks using the imported library:

import { SkyA11y } from '@blackbaud/skyux-builder/runtime/testing/e2e'; SkyA11y.run().then((violations: number) => { expect(violations).toEqual(0); done(); }); Manage accessibility rules

To enable or disable rules for accessibility tests, use the a11y property in the skyuxconfig.json file. You can manage rules individually, or you can enable or disable all rules in bulk. For information about individual rules in aXe automated tests, see the complete list of accessibility rules.

  • To disable individual rules, add the a11y property and its rules property. Then list the rules to disable and set enabled to false:

    { "a11y": { "rules": { "label": { "enabled": false } } } }

  • To enable individual rules that are disabled by default, add the a11y property and its rules property. Then list the rules to enable and set enabled to true:

    { "a11y": { "rules": { "hidden-content": { "enabled": true } } } }

  • To disable all rules, set the a11y property to false:

    { "a11y": false }

  • To enable all rules, including rules that are disabled by default, set the a11y property to true:

    { "a11y": true }
For SKY UX guidance on accessibility best practices, see accessibility recommendations.
