Configuration

This reference describes configuration options for the SKY UX process.

Commands

After you create a SKY UX project, you can use the skyuxconfig.json file in the project directory to configure certain aspects of the SKY UX process. Some configuration options are defined by SKY UX Builder and are included in skyuxconfig.json that comes from running skyux new.

The configuration options that are available in the skyuxconfig.json file include:

  • name — The name of your project when running in SKY UX Host. For example, if the name is "demo," then your SPA is accessible from https://sky.blackbaud-dev.com/demo. The default behavior is to read the name property in your `skyuxconfig.json` file, automatically removing "skyux-spa-".
