This grid component displays a SKY UX-themed grid of data. The grid should contain sky-grid-column components which specify column information. The sky-grid component can be used as a view of a searchable, paginated list component when used within the sky-list component. Specifies an array of data for the grid. Each item in the array should have an id property as well as properties that map to each columns field or id of the grid. Specifies the columns to be displayed by the grid. Will match the id or field of the column. If no array is provided, then all columns are shown. Acceptable values: Array of strings. Specifies how the grid should fit to its parent. Acceptable values: 'width' or 'scroll'. Specifies the height of the grid. Acceptable values: number. Specifies the width of the grid. Acceptable values: number. Fires when columns are changed in the grid. Emits an array of column ids. Specifies a unique id for the column. If no field property exists on the column, then this will be used as the property to retreive cell information from an entry on the grid data array. Specifies the property to retreive cell information from an entry on the grid data array. If no id property exists on the column, then this will be used as the entry for the grid selectedColumnIds array. Specifies text for the column header. Specifies a description for the column. Specifies the width of the column. Specifies whether the column is locked. If set to true, then the column cannot be dragged to another position, and other columns cannot be dragged before the locked column. Specifies whether the column is initially hidden when grid selectedColumnIds are not provided. Specifies a search function to apply for the specific column. By default, the column will execute a string compare on the column data. Acceptable values: (data: any, searchText: string) => boolean Specifies a template for a column. This can be assigned as a reference to the template attribute, or it can be as a child of the template element inside of the sky-grid-column component. This template has access to the value variable, which contains the value passed to the column, and the row variable, which contains the entire row data.
