This first step to work with SKY UX is to create your single-page application. You can create the local SKY UX application with the SKY UX CLI, but we recommend that you first set up a Git repo to store the source files for the SPA. You can create the repo in Visual Studio Team System or GitHub. Then when you create the local SKY UX application, the SKY UX CLI clones the repo and automatically pulls in the SKY UX template.

Internal Blackbaud developers should create repos and provision their applications through the SKY UX App Management Portal before they create local SKY UX applications. Blackbaud developers can access documentation for the SKY UX App Management Portal on the Engineering System website, which is for Blackbaud internal use only and is not available to external developers.

  1. From the command prompt, cd into the directory where you want to create the application.

  2. Enter skyux new. A series of prompts appears.

  3. In the first prompt, specify a root directory for your project. The SKY UX CLI creates a folder with this name and prefixes it with "skyux-spa."

    The root directory determines how to access the SPA because the URL for the home page includes the root directory after the base URL.

  4. In the second prompt, specify the URL for your Git repo to store the source files for your application. You can use a Visual Studio Team System Git repo or a GitHub repo. The SKY UX CLI clones the repo and automatically pulls in the SKY UX template.

    Internal Blackbaud developers should create repos and provision their applications through the SKY UX App Management Portal before they create local SKY UX applications.

    We do not recommend creating the local SKY UX application before the Git repo. However, if the Git repo is not ready and you want to work locally only, you can skip the second prompt. Then when the repo is available, you can manually add it as a remote to your local project. To add the repo as a remote, you enter git remote add origin [URL] in the command prompt, where "[URL]" is the path for your repo. Then to push your local content to the external repo, you enter git push origin --all.

  5. After the SKY UX CLI downloads the SKY UX template project and installs its dependencies, the command prompt displays the directory name: "SPA [directory name] created in directory "skyux-spa-[directory name]."

    For information about the default items that the template provides in a SKY UX single-page application, see the template items reference.
