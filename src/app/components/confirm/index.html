The confirm service launches simple confirmation dialogs to allow users to confirm actions. The SkyConfirmService provider lauches the confirmation dialog. Within the service, you may specify the dialog's message and type, and customize the button text. Opens a confirmation dialog. The config parameter is of type SkyConfirmConfig and passed to the component's constructor. This method returns the confirmation dialog instance. The message to display in the dialog. An enum of type SkyConfirmType that specifies the number of buttons to include in the dialog. An optional array of SkyConfirmButtonConfig that overrides the default button config. (The number of items in this array must match the number provided for the type property.) A one-button dialog with 'OK' as the default button text. The integer value equivalent is 1. A two-button dialog with 'Yes' and 'Cancel' as the default button text. The integer value equivalent is 2. A three-button dialog with 'Yes', 'No', and 'Cancel' as the default button text. The integer value equivalent is 3. Provide the Custom type if you want to define your own buttons. This setting should be accompanied with a buttons config (see below). The button's text. A boolean property that specifies whether to autofocus this button. A string that describes the button's visual appearance. (Default: "default"). A string identifier that is returned after the closed event is emitted. This is useful to determine which button the user clicked. Default values are applied based on the type of dialog being used. The event that the confirmation dialog instance emits after a user action is received. An object is returned that provides information about the button clicked.
