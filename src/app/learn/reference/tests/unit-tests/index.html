SKY UX automatically configures Karma to run unit tests that match the *.spec.ts pattern. We recommend that you follow the Angular 2 naming guidelines on how to name spec files and where to store them. SKY UX unit tests should use the same name as the components or services that they test, but they should end with .spec.ts. And you place the spec files in the same folder as the components or services that they test.

When you create a SKY UX project, the template includes sample unit tests that you can run to test the default single-page-application. From the command line, you can use the skyux test command to run unit tests and the skyux watch command to run unit tests and watch the file system for changes.

For tips and techniques on testing Angular applications, see the Angular 2 testing guidelines.
