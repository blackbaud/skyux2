This reference provides guidance on unit tests and end-to-end tests in SKY UX projects. The continuous integration layer of SKY UX runs unit tests, visual regression tests, and end-to-end tests in multiple platforms and browsers when the SPA code changes or when an SPA dependency releases a new version. It also handles staging the SPA for additional testing and validation.

 Unit tests

SKY UX automatically configures Karma to run unit tests in the src/app folder that match the .spec.ts pattern. We recommend that you follow the Angular naming guidelines on how to name spec files and where to store them. SKY UX unit tests should use the same name as the components or services that they test, but they should end with .spec.ts. And you place the spec files in the same folder as the components or services that they test. When you run skyux test or skyux watch, code coverage results are stored in the coverage folder.

When you create a SKY UX project, the template includes sample unit tests that you can run to test the default single-page-application. From the command line, you can use the skyux test command to run unit tests and the skyux watch command to run unit tests and watch the file system for changes.

SKY UX styles are loaded when running unit tests, so CSS rules defined in SKY UX take effect during tests. This allows you to check the expected computed style of an element when using things like the HTML hidden property that only take effect when SKY UX styles are loaded.

For tips and techniques on testing Angular applications, see the Angular testing guidelines. And for information about how to test modals in SKY UX, see write unit tests for modals. End-to-end (e2e) tests

SKY UX automatically configures Protractor to run end-to-end tests in the e2e folder that match the .e2e-spec.ts pattern. We recommend that you follow the Angular naming guidelines on how to name spec files and where to store them. SKY UX end-to-end tests should use the same name as the components or services that they test, but they should end with .e2e-spec.ts. Unlike unit tests, you should not place these spec files in the same folder as the components or services that they test. Instead, you should place them in the e2e folder in the root of your project.

When you create a SKY UX project, the template includes a sample end-to-end test that you can run to test the default single-page application. From the command line, you can use the skyux e2e command to run end-to-end tests.

For tips and techniques on testing Angular applications, see the Angular testing guidelines.
