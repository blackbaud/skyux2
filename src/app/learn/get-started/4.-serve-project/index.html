After you create a SKY UX project, you can serve the default single-page application. For more information about skyux serve and other SKY UX CLI commands, see the CLI commands reference.

 Since skyux serve serves your single-page application over HTTPS, any additional requests from your code such as AJAX requests must also be over HTTPS.

  1. From the command prompt, cd into the root directory for the SKY UX project from the previous tutorial.

  2. Enter skyux serve to launch the SPA in your default browser. This opens the host endpoint and instructs SKY UX Host to pull files from the local machine instead of the CDN.

    By default, skyux serve launches the Host URL, which means that it reaches out to SKY UX Host for an HTML page to host your local files. However, you can use the -l or --launch options to specify the URL to launch.

    • skyux serve --launch local — Launches the local URL. This creates a local HTML page that is roughly the same as the HTML file that SKY UX Host builds for production. This option is useful for serving the SPA when you don't have Internet access.

    • skyux serve --launch none — Stops any URL from launching. This option is useful if you want to serve the SPA but do not want it to automatically open in the web browser.

    • skyux serve --launch host — (Default.) Launches the Host URL. This option provides the closest test to production because when skyux serve creates a local web server to launch the SPA, it reaches out to SKY UX Host for an HTML page to host your local files.

    By default, the SPA automatically opens in the default browser, but you can specify a different browser with the --browser option. This option is useful for cross-browser manual testing, and the available values are chrome, edge, firefox, iexplore, and safari. For example, skyux serve --browser firefox.

  3. In the command prompt, skyux serve continues to run, and changes to your project are automatically displayed as you save them.
