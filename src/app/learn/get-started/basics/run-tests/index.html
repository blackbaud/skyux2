After you create a SKY UX project, serve the default single-page application, and edit pages and components in the project, you can follow these tutorials to run sample tests for the default SPA.

 Run sample unit test

Unit tests scrutinize the smallest testable parts of an application. When you create a SKY UX project, the template includes sample unit tests in the new project. You can follow the steps in this tutorial to run the unit test for the default single-page application.

For information about unit tests in SKY UX, see unit tests, and for information about how to test modals in SKY UX, see write unit tests for modals. For additional tips and techniques on testing Angular applications, see the Angular testing guidelines.

  1. To view the unit test spec files in a local editor, open the project's about.component.spec.ts file in the src/app/about folder and the home.component.spec.ts file in the src/app folder.

    SKY UX unit tests should use the same name as the components or services that they test, but they should end with .spec.ts. The spec files should also be in the same folder as the components or services that they test.

  2. In the command prompt, if skyux serve is still running, enter Ctrl-C to stop serving the project.

  3. Enter skyux test to run the unit tests.

  4. The tests run, and the command prompt displays the results.

    When you run skyux test or skyux watch, code coverage results are stored in the coverage folder.
Run sample end-to-end test

End-to-end tests identify system dependencies and ensure that an application flows as expected from start to finish. When you create a SKY UX project, the template includes a sample end-to-end test in the new project. You can follow the steps in this tutorial to run the end-to-end test for the default single-page application.

For information about end-to-end tests in SKY UX, see end-to-end tests. And for tips and techniques on testing Angular applications, see the Angular testing guidelines.

  1. To view the end-to-end test spec file in a local editor, open the project's about.component.e2e-spec.ts file in the e2e folder at the root of the project.

    SKY UX end-to-end tests should use the same name as the components or services that they test, but they should end with .e2e-spec.ts. Place spec files for end-to-end tests in the e2e folder in the root of your project.

  2. In the command prompt, if skyux serve is still running, enter Ctrl-C to stop serving the project.

  3. Enter skyux e2e to run the end-to-end test.

  4. The test runs, and the command prompt displays the results.

Controlling Test Runs

During development there may be times when you want control over which tests are included during a test run. You can follow the steps in this tutorial to better control your test runs.

Exclude tests from a test run

In order to exclude tests from a test run you can simply change it to xit for the test you wish to exclude or describe to xdescribe if you wish to exclude an entire test suite.

Explicitly include tests in a test run

If you would like to limit a test run to only include a subset of tests you can change it to fit or describe to fdescribe and only those tests or test suites will be included in the test run.
