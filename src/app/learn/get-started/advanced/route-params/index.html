In addition to automatically registering routes and creating components for all index.html files in the src/app directory, SKY UX supports routes that contain route parameters.

To create a route parameter, you create a subfolder and prefix the name with an underscore. The string after the underscore is used as the variable name. For example, to use a route parameter in src/app/dogs, you can create a src/app/dogs/_id folder and then create an index.html file with the following content:

You are viewing dog {{ id }}.

When users visit the route dogs/42, the id variable is set to "42" and renders as:

You are viewing dog 42.
